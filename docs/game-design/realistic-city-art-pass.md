# 写实化城市场景美术规格

更新时间：2026-06-10

## 目标

本轮目标是把 MVP 从抽象几何块推进到移动端可上线截图级的半写实 3D 城市。风格不追求照片级高模，而是对标移动超休闲竞品的清晰体量、丰富轮廓和快速识别：玩家在俯视相机下能立刻分辨行人、汽车、公交、路灯、树木、房屋、街道和特殊建筑槽位。

## 材质策略

- 使用 Unity 内置 Standard Shader，保持 Android/Google Play 首发构建稳定。
- 通过金属度和光滑度区分混凝土、金属、玻璃、橡胶、路面和植被。
- 发光材质只用于车灯和路灯灯体，默认不生成大量实时点光源，保护中低端安卓帧率。
- 继续保留高饱和识别色：公交黄、消防红、警局蓝、道路黄线，避免写实后可读性下降。

## 光照与环境

- 主光使用软阴影，补光降低黑面死角。
- 环境光使用 Trilight，让天空、地平线、地面有不同色温。
- 远景启用线性雾，弱化无限区块边界和对象销毁感。
- 相机使用纯色天空背景、48 度 FOV、220 米远裁剪，适配竖屏俯视玩法。

## 性能边界

- 写实细节全部是程序化低模几何，装饰件移除 Collider，只保留可吸收对象根 Collider。
- 当前激活半径仍为 2，常驻约 25 个区块；新增细节前优先控制每区块对象数。
- 下一轮需要引入对象池和合批策略，优先处理车辆、路面装饰、窗户、树冠这些重复对象。
- Meta Quest 版本需要单独降低雾和阴影成本，并评估 URP/Single Pass Instanced。

## 验收标准

- 城市截图中每类对象不用文字也能识别。
- 开局 10 秒内可见至少 3 类 T1/T2 可吸收目标。
- 远离玩家的区块销毁不会出现明显空洞。
- Android Debug APK/AAB 构建成功，EditMode 测试通过。
